# Phase 8, Plan 1: DAG Optimizer Module

**Goal:** Build DAG optimizer module with topological sort optimization, ASCII visualization, interactive review, and CLI integration.

## Tasks

### 1. `dag_optimizer/engine.py` — DAG Engine
- `build_dag(steps)` → dict of `{step_name: set(child_names)}` parent→children mapping
- `build_reverse_dag(steps)` → dict of `{step_name: set(parent_names)}` (from Step.parents)
- `compute_optimal_orders(steps)` → `{step_name: new_order}` via BFS topological level assignment
- `apply_optimization(flow)` → new Flow with updated Step.order values, preserving all other fields
- `suggest_intermediate_steps(flow, threshold=3)` → detect steps with >N children, return suggestions
- `render_ascii_dag(steps, title, highlight_diffs=None)` → Rich renderable ASCII DAG with levels

### 2. `dag_optimizer/review.py` — Interactive Review
- `review_optimization(console, flow, optimized_flow, suggestions)` → interactive prompt
- Shows before/after DAG side by side or sequentially
- Prompts user: accept all / reject all / custom (accept specific changes)
- Returns: accepted Flow or original Flow

### 3. CLI Integration (`cli.py`)
- Add `--optimize / --no-optimize` flag to `generate`, `interactive`, `batch` commands
- Default: `--optimize`
- Wire into `run_evaluation_loop()` in `pipeline.py`
- In auto mode: apply optimization silently, log summary
- In interactive mode: call `review_optimization()`

### 4. Pipeline Integration (`pipeline.py`)
- After `write_flow_json()`, if optimize=True, run optimizer
- If auto mode: apply directly
- If interactive: show review UI
- Re-write JSON if optimization changed anything

### 5. Tests
- `tests/test_dag_optimizer.py`
- Test: build_dag returns correct parent→child mapping
- Test: compute_optimal_orders assigns levels correctly (root=0, chain, diamond)
- Test: apply_optimization preserves non-order fields
- Test: suggest_intermediate_steps detects fan-out
- Test: `--no-optimize` passes through unchanged
- Test: render_ascii_dag returns expected output structure
- Test: CLI flags parse correctly
