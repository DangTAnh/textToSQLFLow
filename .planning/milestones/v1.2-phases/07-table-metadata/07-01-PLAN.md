# 07-01-PLAN: Table Metadata Parser + Prompt Integration

**Phase:** 7 â€” Table Metadata
**Goal:** Parse table metadata from JSON/DDL files and inject into LLM prompt
**Mode:** standard (single wave)

## Tasks

### 1. Pydantic Models (`table_metadata/models.py`)

- `ColumnMetadata`: name, type, description, nullable, is_key, partition_column
- `TableMetadata`: name, description, columns: list[ColumnMetadata], partitioned_by: list[str]
- `TableMetadataFile`: tables: list[TableMetadata] (root model for JSON array)

### 2. DDL Parser (`table_metadata/ddl_parser.py`)

- `parse_ddl(text: str) -> list[TableMetadata]`
- Regex patterns: `CREATE TABLE` header, column lines (`name TYPE`), constraints
- Handle: `PRIMARY KEY (col)`, `PARTITIONED BY`, `COMMENT '...'`, `NOT NULL`
- Skip: indexes, storage clauses, `USING`, `WITH`

### 3. JSON Parser (`table_metadata/parser.py`)

- `parse_table_metadata_file(path: Path) -> list[TableMetadata]`
- Auto-detect format: `.json` â†’ JSON parse â†’ Pydantic validate; `.sql`/`.ddl` â†’ text read â†’ DDL parse
- Unknown extension â†’ try JSON first, fallback DDL
- Error handling: per-table failure = warn + skip

### 4. Prompt Integration (`llm/prompts.py`)

- `build_generation_prompt(description, table_metadata=None)` â€” new optional param
- When metadata provided: inject section after business description:
  ```
  ## Available Tables
  - invoice (columns: invoice_id string PK, amount decimal, ...)
  ```
- `--tables-include-ddl` â†’ inject full DDL text instead of summary

### 5. CLI Integration (`cli.py`, `pipeline.py`)

- `generate` command: add `--tables` (Path | None), `--tables-include-ddl` (bool)
- `interactive` command: add `--tables`
- `pipeline.py`: pass metadata through to prompt builder
- `interactive.py`: optionally prompt user for tables file

### 6. Tests (`tests/test_table_metadata.py`)

- JSON parse valid file â†’ correct TableMetadata
- DDL parse â†’ correct columns, types, keys, partitions
- Auto-detect JSON vs DDL by extension
- Auto-detect unknown extension (try JSON â†’ fallback DDL)
- File not found â†’ clear error
- Malformed JSON â†’ ValueError
- DDL with unsupported syntax â†’ partial parse, warn
- Prompt builder with metadata â†’ output contains table names
- Prompt builder without metadata â†’ unchanged
