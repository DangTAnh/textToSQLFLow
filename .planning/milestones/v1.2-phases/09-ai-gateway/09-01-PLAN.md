# Phase 9, Plan 1: AI GATEWAY Service + CLI Integration

**Goal:** Build standalone FastAPI proxy service and wire CLI to optionally use it.

## Files to Create

### `gateway/main.py` — FastAPI entry point
- `POST /v1/chat/completions` — proxy endpoint
- `GET /health` — health check
- `GET /v1/models` — list models from config
- Middleware: audit logging (log request/response metadata)
- Startup: load config, init cache, rate limiter, cost tracker

### `gateway/config.py` — Pydantic models for gateway.yaml
- GatewayConfig, RoutingRule, FallbackConfig, RateLimitConfig, CacheConfig, AuditConfig, RBACEntry
- `load_gateway_config()` function

### `gateway/models.py` — Request/Response models
- ChatCompletionRequest (OpenAI-compatible: model, messages, temperature, etc.)
- ChatCompletionResponse, Choice, Usage

### `gateway/llm.py` — Core handler
- `route_request(config, prompt_text) → (provider, model)` — regex match
- `call_llm_with_fallback(config, provider, model, messages) → (response, provider_used, cost)`
- Cost estimation based on token counts

### `gateway/cache.py` — In-memory cache
- `ResponseCache(max_size, default_ttl)` — dict-based with TTL
- `get(key)`, `set(key, value, ttl)`, `invalidate(pattern)`

### `gateway/rate_limiter.py` — Token bucket rate limiter
- `RateLimiter(rates: dict[str, int])` — per-provider RPM
- `check(provider) → (allowed, retry_after)`

### `tests/test_gateway.py`
- Test: config loading
- Test: routing (regex pattern matching)
- Test: cache (get/set, TTL expiry)
- Test: rate limiter (allow/deny, reset)
- Test: cost tracking

### `tests/test_gateway_e2e.py` (Phase 10, integration)
- Test: FastAPI TestClient endpoint
