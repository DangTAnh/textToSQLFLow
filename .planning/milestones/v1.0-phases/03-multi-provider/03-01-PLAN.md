    Create text_to_sql_flow/config.py with the following structure:
    
    ```python
    from pydantic import BaseModel, Field
    from typing import Optional
    from pathlib import Path
    
    class AppConfig(BaseModel):
        """Application configuration loaded from YAML file with env var fallback."""
        provider: str = "openai"
        api_key: Optional[str] = None
        model_name: Optional[str] = None
        temperature: float = 0.3
        max_tokens: Optional[int] = None
    
    def load_config(config_path: Optional[Path] = None) -> AppConfig:
        """Load config from YAML file, falling back to env vars.
        
        Resolution priority:
        1. CLI flags (handled by caller, passed to run_generation)
        2. Config file values
        3. Environment variables
        4. Defaults
        """
    
    def _resolve_api_key(config: AppConfig, provider: str) -> str:
        """Resolve API key: use config.api_key if set, else read from provider-specific env var.
        
        Maps: openai->OPENAI_API_KEY, claude->ANTHROPIC_API_KEY,
              deepseek->DEEPSEEK_API_KEY, nvidia->NVIDIA_API_KEY,
              openrouter->OPENROUTER_API_KEY, opencode->OPENCODE_API_KEY
        """
    ```

    Config file format (text-to-sql-flow.yaml):
    ```yaml
    provider: openai
    api_key: sk-...
    model_name: gpt-4o
    temperature: 0.5
    max_tokens: 4096
    ```
    
    Key behaviors:
    - Default config path: ./text-to-sql-flow.yaml (per D-03 from CONTEXT.md)
    - If config file doesn't exist, return default AppConfig (graceful -- not an error)
    - If config file exists but is malformed YAML, raise ValueError
    - Pydantic validates all fields: provider must be a valid string, temperature between 0 and 2
    - _resolve_api_key: if config.api_key is provided, use it. Otherwise read from the provider-specific env var. Raise ValueError if no key found in either location.
    - model_name defaults to None, meaning "use provider default" (caller maps to litellm model)
    - All Pydantic validation is done on load, not lazily

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML config file -> config.py | Untrusted YAML content from filesystem read by application |
| provider.py -> litellm -> external API | Provider model names and API keys sent to external LLM APIs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | config.py YAML load | mitigate | Pydantic validates all fields after load; malformed YAML raises ValueError, not arbitrary code exec (PyYAML safe_load is the default in recent versions -- use yaml.safe_load not yaml.load) |
| T-03-02 | Information Disclosure | config.py API key | mitigate | API key read at call time, never logged (check logger doesn't print api_key value), never written to output files |
| T-03-03 | Tampering | provider.py litellm import | mitigate | litellm is an established package. Use explicit `model=` string mapping to prevent injection |
| T-03-SC | Tampering | pip dependencies | mitigate | litellm >=1.40 [ASSUMED], PyYAML >=6.0 [ASSUMED], Jinja2 >=3.1 [ASSUMED] per fallback policy. All are long-established packages with wide adoption |
