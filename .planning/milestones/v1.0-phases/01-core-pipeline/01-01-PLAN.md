    Create text_to_sql_flow/types.py with these Pydantic v2 models matching sample.txt exactly (per decision D-11, D-12):

    ```python
    from datetime import datetime
    from pydantic import BaseModel, Field
    from typing import Optional

    class CreatedDate(BaseModel):
        """Wraps the $date format used in sample.txt"""
        date: datetime = Field(alias="$date")

    class StepOutput(BaseModel):
        """steps.output — stores execution result info"""
        tempView: str
        table: str
        appendType: str = "REPLACE"
        kafkaGroup: str = ""

    class Diagram(BaseModel):
        """steps.diagram — display position on the flow diagram"""
        x: int
        y: int

    class Step(BaseModel):
        """Single step in the Spark SQL flow"""
        name: str
        parents: list[str] = Field(default_factory=list)
        order: int
        sql: str
        output: StepOutput
        description: str = ""
        diagram: Diagram
        active: bool = True
        createdDate: CreatedDate

    class Flow(BaseModel):
        """Top-level flow model — the complete Spark SQL ETL flow"""
        name: str
        description: str = ""
        steps: list[Step]
    ```

    Use `model_config = ConfigDict(populate_by_name=True)` on each model so both field names and aliases work.
    Set `CreatedDate` as a nested model for the `{"$date": "..."}` format — Pydantic v2 handles this via `model_validate`.
    Add a helper method `Flow.to_serializable_dict()` that serializes `createdDate.$date` back to `{"$date": "..."}` format for JSON output.

    Include docstrings on every class describing the field semantics from sample.txt (e.g., "tempView: spark temp view name, data cached on storage").

    Create text_to_sql_flow/cli.py using Typer (per D-01):

    ```python
    import typer
    from pathlib import Path
    from typing import Optional

    app = typer.Typer(
        name="text-to-sql-flow",
        help="Generate Spark SQL ETL flows from business descriptions using LLM",
    )

    @app.command()
    def generate(
        description: str = typer.Argument(
            ...,
            help="Business description of the Spark SQL ETL flow",
        ),
        output: Optional[Path] = typer.Option(
            "./output",
            "--output",
            "-o",
            help="Output directory for generated flow files",
            file_okay=False,
            dir_okay=True,
        ),
    ):
        \"\"\"
        Generate a Spark SQL flow from a business description.

        Calls an LLM to analyze the description and produce a structured
        Spark SQL ETL flow definition, then writes the result as a JSON file.
        \"\"\"
        # Lazy import — pipeline module is created in Plan 02
        from text_to_sql_flow.pipeline import run_generation

        result_path = run_generation(description=description, output_dir=output)
        typer.echo(f"Flow generated successfully: {result_path}")
    ```

    Importing `pipeline` inside the function body (lazy) means the module must exist at call-time but not at import-time. This allows Plan 01 to ship without pipeline.py existing yet.

    Do NOT add --help customization, rich formatting, or progress bars — those belong in later phases.
