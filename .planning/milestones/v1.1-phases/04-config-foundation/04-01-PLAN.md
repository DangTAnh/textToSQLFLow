# 04-01-PLAN: .env Loading + Default Provider

**Phase:** 4 — Config Foundation
**Goal:** Add `.env` file support and switch default provider to opencode
**Mode:** quick (single wave, no sub-phases)

## Tasks

### 1. Add `.env` loader to `config.py`

- Create `_parse_dotenv(path: Path) -> dict[str, str]` — manual parser
  - Read file, skip blank lines and `#` comment lines
  - Split on first `=`, strip whitespace, handle quoted values (strip surrounding `"` or `'`)
  - Return dict of key→value
- Create `load_dotenv(path: Optional[Path] = None)` — public function
  - Default path: `./.env` in current directory
  - Parse file, merge into a module-level `_dotenv_values` dict
  - Call once at import time (like `python-dotenv` does)

### 2. Update `resolve_api_key()` priority

New priority chain:
1. Check `_dotenv_values` (from `.env` file)
2. Check `os.environ` (system env vars)
3. Check `config.api_key` (YAML config)
4. Raise `ValueError`

### 3. Change default provider → `opencode`

- `config.py`: `AppConfig.provider = "opencode"`
- `cli.py`: `provider: str = typer.Option("opencode", ...)`
- `pipeline.py`: both `run_generation()` and `run_evaluation_loop()` defaults
- `llm/provider.py`: `call_llm(provider="opencode")`
- `evaluator.py`: `evaluate_flow(provider="opencode")`

### 4. Tests

- Create `tests/test_config.py`
  - `test_load_dotenv_parses_simple`
  - `test_load_dotenv_skips_comments_and_blanks`
  - `test_load_dotenv_handles_quoted_values`
  - `test_resolve_api_key_priority_dotenv_over_env`
  - `test_resolve_api_key_priority_env_over_config`
  - `test_default_provider_is_opencode`
